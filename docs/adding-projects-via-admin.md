# Как добавить проект на сайт через админку Cultnova

Пошаговое руководство по публикации новых проектов. Следуйте этой инструкции, чтобы корректно заполнить все данные, оптимизировать страницу для SEO и избежать ошибок отображения.

## Краткий алгоритм

1. Проверьте или создайте категорию.
2. Добавьте проект и заполните текстовые поля.
3. Загрузите обложку.
4. Настройте SEO.
5. Добавьте медиафайлы (галерею).
6. Проверьте данные, сохраните и опубликуйте.

---

## 1. Выберите или создайте категорию

Путь: **Проекты** → **Категории проектов** → **Добавить**

*Если нужная категория уже существует, пропустите этот шаг.*

Заполните поля:
*   **Название** — имя категории на сайте.
*   **Слаг** — часть URL-адреса.

💡 **Совет:** Пишите слаг латиницей в нижнем регистре, слова разделяйте дефисом (например: `web-design`).

## 2. Добавьте проект и основную информацию

Путь: **Проекты** → **Проекты** → **Добавить**

Заполните обязательные поля:
*   **Название** — имя проекта (оно же станет главным заголовком страницы `H1`).
*   **Слаг** — часть адреса страницы (например: `project-13`).
*   **Категория** — выберите подходящую из списка.
*   **Заказчик** — название компании или клиента.
*   **Год** — год реализации проекта.
*   **Тип проекта** — короткое описание формата работ.
*   **Excerpt** — краткий анонс. Он выводится на карточке проекта в общем списке и в блоке «Похожие проекты».
*   **Body HTML** — основной текстовый контент страницы.

⚠️ **Важно по тексту (Body HTML):**
*   Не добавляйте заголовок `H1` внутрь текста, так как им автоматически становится **Название** проекта. Начинайте структуру с `H2`.
*   Если в тексте нужны изображения, вставляйте их прямо через редактор этого поля.

## 3. Загрузите обложку

В блоке **Preview** настройте карточку проекта:

*   **Upload preview image** — загрузите главную обложку. Она появится в общем списке проектов и в начале самой страницы.
*   **Preview image alt** — кратко опишите, что изображено на обложке (обязательно для поисковиков).

## 4. Заполните SEO-настройки

В блоке **SEO** укажите данные для поисковых систем:

*   **SEO title** — заголовок для поисковиков и вкладки браузера (до 60 символов).
*   **SEO description** — краткое описание страницы (до 160 символов).
*   **SEO keywords** — ключевые слова (по необходимости).
*   **SEO robots** — по умолчанию оставьте `index, follow`.
*   **Canonical URL** — заполняйте, только если настраиваете канонический адрес.

💡 **Совет:** Если страницу проекта нужно скрыть от поисковиков (например, это черновик для клиента), укажите в поле **SEO robots** значение `noindex, nofollow`.

## 5. Добавьте галерею (Медиа-блоки)

В самом низу настроек проекта находится блок **Project media**.

Нажмите **Add another Project media**. Для сборки галереи используйте только два типа блоков: **image** (фото) и **video** (видео). Текстовые блоки здесь не используются (весь текст задается в Body HTML).

### Добавление изображения
*   **Type** — выберите `image`.
*   **Order** — цифра, определяющая порядок показа на странице (1, 2, 3...).
*   **Media** — загрузите файл или укажите ссылку.
*   **Media alt** — обязательно опишите картинку.
*   **Caption** — подпись под фото (если нужна).

### Добавление видео
*   **Type** — выберите `video`.
*   **Order** — порядок показа.
*   **Media** — загрузите файл или укажите ссылку.
*   **Upload first frame** — **обязательно** загрузите картинку (первый кадр/превью), которая будет показываться до запуска видео.
*   **Caption** — подпись под видео (если нужна).

## 6. Проверка, сохранение и публикация

Перед тем как нажать **Сохранить**, быстро пробегитесь по чек-листу:
- [ ] Категория выбрана.
- [ ] Слаг написан латиницей и уникален.
- [ ] Body HTML заполнен (без дублирования H1).
- [ ] SEO-теги прописаны.
- [ ] У всех картинок (включая обложку) заполнен `alt`.
- [ ] У всех видео загружен первый кадр (Upload first frame).
- [ ] Порядок медиа-блоков (Order) выставлен логично.

### Управление публикацией
*   Проект автоматически появляется на сайте сразу после сохранения (если стоит галочка **Опубликовано**).
*   Страница будет доступна по адресу `/projects/<slug>/`. В админке можно кликнуть **Public URL**, чтобы сразу перейти на сайт.
*   Если вы хотите снять проект с публикации, **уберите галочку «Опубликовано»** и сохраните. Страница пропадет с сайта и из sitemap.

---

## ❌ Частые ошибки (шпаргалка)

*   Забыли заполнить **Body HTML** (страница будет пустой).
*   Оставили пустыми поля **alt** у изображений.
*   Не загрузили обложку для видео (останется черный экран).
*   Написали неуникальный **Слаг** (система может выдать ошибку).
*   Добавили заголовок **H1** внутрь текста (нарушение SEO).
*   Опубликовали проект до того, как он полностью согласован и готов к показу.

*Если после сохранения страница на сайте не обновилась или отображается некорректно — сообщите нам, мы проверим пересборку проекта на сервере.*
